INSTRUCCIONES - Ejecutar el scanner y parser (Lenguaje Caminante)

Resumen
-------
Este documento explica, paso a paso, cómo preparar el entorno y ejecutar el scanner (lexer) y el parser (yacc) que están en este repositorio.

Requisitos
----------
- Windows (PowerShell). Estas instrucciones usan PowerShell.
- Python 3.8+ instalado y accesible vía `python` en la consola.
- Dependencias del proyecto: están listadas en `requirements.txt`.

Pasos
-----
1) Abrir PowerShell y situarse en la raíz del repositorio (donde está este archivo):

   Set-Location -Path "C:\Users\sebas\Documents\UNaHur\Parseo\Parseo"

2) Instalar dependencias (si no están ya instaladas):

   python -m pip install --user -r requirements.txt

   Nota: `requirements.txt` ya incluye `ply`. Si prefieres instalar globalmente, quita `--user`.

3) Crear un archivo de ejemplo con un programa mínimo (opcional):

   @"
   inicio
   mover norte
   fin
   "@ | Out-File -Encoding utf8 sample.cam

   Esto crea `sample.cam` con un programa simple.

4) Ejecutar sólo el scanner (ver tokens):

   python -c "from caminante_lex import lexer; data=open('sample.cam','r',encoding='utf-8').read(); lexer.input(data); print('--- TOKENS ---');
import sys
for tok in lexer: print(tok)"

   Esto imprimirá los tokens reconocidos por el lexer.

5) Ejecutar sólo el parser (parsear el contenido):

   python -c "from caminante_yacc import parse; data=open('sample.cam','r',encoding='utf-8').read(); ok = parse(data); print('Parse OK:', ok)"

   La función `parse` devuelve `True` si no hubo errores sintácticos.

6) Ejecutar scanner y parser en conjunto (ejemplo rápido):

   - Para ver tokens y luego parsear en un mismo comando:

   python -c "from caminante_lex import lexer; from caminante_yacc import parse; data=open('sample.cam','r',encoding='utf-8').read(); print('TOKENS:'); lexer.input(data);
for t in lexer: print(t); print('\nPARSING:'); print('Parse OK:', parse(data))"

   - Alternativamente, ejecutar paso a paso: primero ver tokens (paso 4), luego parsear (paso 5).

7) Ejecutar la suite de tests unitarios (verificar que todo funciona):

   python -m unittest discover -s tests -v

   Esto ejecuta los tests incluidos en `tests/`.

Notas y recomendaciones
----------------------
- Ejecuta los comandos desde la raíz del repo. Los módulos (por ejemplo `caminante_lex.py`, `caminante_yacc.py`) se importan directamente por nombre.
- Si decides ejecutar desde otra carpeta, añade la raíz del repo al `PYTHONPATH` o usa `Set-Location` para situarte en la raíz.
- Si ves warnings de PLY sobre tokens sin uso o conflictos *shift/reduce*, son informativos (aparecen durante la generación de tablas) y no impiden las ejecuciones de ejemplo ni los tests, pero pueden indicar definiciones no usadas en el lexer o ambigüedades en la gramática.
- `parsetab.py` se genera automáticamente por PLY si no existe. En este repo hay una copia en `Código/parsetab.py` (respaldo). Si quieres forzar re-generación, borra `parsetab.py` y vuelve a ejecutar una invocación que construya el parser (por ejemplo, importar `caminante_yacc`).
